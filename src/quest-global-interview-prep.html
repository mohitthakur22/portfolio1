Frontend Core
Explain the difference between display: flex and display: grid. When do you choose one over the other?
⌄
CSS · Layout High Priority

Flexbox is one-dimensional — it controls flow along a single axis (row or column). Great for navbars, button groups, card rows, and anything where items just need to line up.

Grid is two-dimensional — it gives you explicit control over rows AND columns simultaneously. Ideal for page layouts, dashboards, image galleries.

Rule of thumb: If your layout is driven by content → Flexbox. If content needs to fit into a defined layout → Grid.

★ Interviewer TipDraw a quick ASCII sketch of each. Showing you can reason spatially is impressive.
How do you make a web application responsive? Walk me through your approach.
⌄
CSS · Responsive High Priority

Start with a mobile-first CSS approach (min-width media queries). Use relative units (%, rem, vw/vh) instead of px for layout. Apply CSS Grid/Flexbox for fluid layouts. Use clamp() for fluid typography. Test across real devices, not just browser DevTools.

/* Mobile-first */
.card { font-size: clamp(1rem, 2.5vw, 1.25rem); }

@media (min-width: 768px) { .grid { grid-template-columns: repeat(3, 1fr); } }
★ Interviewer TipMention Quest Global likely serves enterprise dashboards — bring up how you handle data-heavy tables or charts on mobile.
Explain event bubbling, event capturing, and stopPropagation() in JavaScript.
⌄
JavaScript High Priority

Bubbling: Events travel from the target element UP through its ancestors (default). Capturing: Events travel DOWN from the document root to the target. Pass true as the third argument to addEventListener to use capturing.

stopPropagation() halts further propagation. stopImmediatePropagation() also prevents other listeners on the same element.

parent.addEventListener('click', () => console.log('parent'), true); // capture
child.addEventListener('click', (e) => {
  e.stopPropagation();
  console.log('child only');
});
★ Interviewer TipMention event delegation as a practical use of bubbling — attaching one listener to a parent instead of many children.
What is the virtual DOM and how does reconciliation work in React?
⌄
React High Priority

The Virtual DOM is a lightweight in-memory representation of the real DOM. When state changes, React creates a new virtual DOM tree and diffs it against the previous one (reconciliation). Only the minimal set of real DOM mutations is applied.

React's diffing algorithm operates in O(n) with two heuristics: (1) elements of different types produce different trees; (2) key props help identify list items across renders.

★ Interviewer TipMention React 18's concurrent rendering and how fiber architecture allows interruptible rendering for better UX.

⌄

Key strategies: Code splitting with React.lazy + Suspense. Memoization with React.memo, useMemo, useCallback. Virtualized lists for large datasets (react-window). Lazy loading images. Profiler to identify bottlenecks. Avoid unnecessary re-renders by lifting state correctly.

const MemoComp = React.memo(({ data }) => );
const sorted = useMemo(() => data.sort(...), [data]);

Node.js & Backend
Explain the Node.js event loop. How does it handle asynchronous operations?
⌄
Node.js High Priority

Node.js is single-threaded but non-blocking. The event loop continuously checks the call stack, and when empty, processes callbacks from the queue. Async I/O (file, network) is offloaded to libuv's thread pool.

Phases: timers → pending callbacks → idle → poll → check (setImmediate) → close callbacks. Microtasks (Promises) run between each phase.

★ Interviewer TipBe ready to explain why CPU-bound tasks block the event loop and how worker threads solve that.
How would you design a REST API for a microservices-based application?
⌄
Node.js · Microservices High Priority

Key principles: Stateless endpoints, proper HTTP verbs (GET/POST/PUT/DELETE), meaningful status codes, versioning (/api/v1/), pagination, and consistent error envelopes. In a microservices setup: each service owns its domain, communicates via REST or events (Kafka/RabbitMQ), has an API Gateway for routing.

GET    /api/v1/users        → list
POST   /api/v1/users        → create
GET    /api/v1/users/:id    → detail
PATCH  /api/v1/users/:id    → update
★ Interviewer TipMention OpenAPI/Swagger for documentation and contract-first design. Quest values structured engineering.
AWS, Docker & DevOps
What AWS services would you use to deploy a React + Node.js application? Walk me through the architecture.
⌄
AWS High Priority

Frontend: S3 (static hosting) + CloudFront (CDN + HTTPS). Backend: ECS/EKS for containers, or Lambda for serverless APIs. Database: RDS (relational) or DynamoDB (NoSQL). CI/CD: CodePipeline or GitHub Actions. Secrets: AWS Secrets Manager. DNS: Route 53.

★ Interviewer TipMention IAM roles and least-privilege access. Security awareness stands out at engineering firms.
What is Helm and why would you use it over plain Kubernetes manifests?
⌄
Helm · Kubernetes Medium Priority

Helm is a package manager for Kubernetes. It bundles K8s YAML manifests into reusable, versioned charts with template variables. Benefits: avoid YAML duplication, manage environment configs with values.yaml, rollback deployments, share charts across teams.

helm install my-app ./chart --set image.tag=v2.1.0
helm rollback my-app 1  # revert to revision 1
★ Interviewer TipIf you've used it, mention Helm in CI/CD pipelines (GitHub Actions → helm upgrade). If not, be honest but show conceptual understanding.
Explain Docker's layered file system and why it matters for build performance.
⌄
Docker Medium Priority

Docker images are built from layers — each RUN/COPY/ADD instruction creates an immutable layer. Layers are cached, so if a layer hasn't changed, Docker reuses it. This is why you should copy package.json and run npm install BEFORE copying app source — the node_modules layer is cached unless dependencies change.

COPY package*.json ./          # cache this layer
RUN npm ci
COPY . .                       # invalidates only from here
★ Interviewer TipMention multi-stage builds to keep production images lean (builder stage vs runtime stage).
